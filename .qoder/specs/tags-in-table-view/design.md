# Отображение тегов в виде бейджей в таблице публикаций - Дизайн документ

## Обзор
Цель этого изменения - реализовать отображение тегов в виде бейджей в таблице публикаций на главной странице, аналогично тому, как это сделано в карточках. В настоящее время в компоненте `PaginatedPostsTable` теги отображаются как обычные текстовые элементы без использования компонента `Badge`.

## Техническая архитектура
Решение будет включать в себя обновление компонента `PaginatedPostsTable` для использования компонента `Badge` из `@/components/ui/badge` для отображения тегов, аналогично тому, как это реализовано в компоненте `PostsTable`.

## Дизайн компонентов
### PaginatedPostsTable
Компонент `PaginatedPostsTable` будет обновлен для использования компонента `Badge` для отображения тегов. Стилизация бейджей будет соответствовать стилю, используемому в компоненте `PostsTable`.

## Модели данных
Не требуется изменений в моделях данных.

## Спецификации API
Не требуется изменений в API.

## Обработка ошибок
Не требуется специальной обработки ошибок для этого изменения.

## Стратегия тестирования
- Проверить, что теги отображаются в виде бейджей в таблице публикаций
- Проверить, что стилизация бейджей соответствует стилю карточек
- Проверить, что ограничение на количество отображаемых тегов (3 + счетчик) сохраняется
- Проверить, что функциональность поиска и фильтрации по тегам не нарушена

## Примечания по реализации
- Необходимо использовать существующий компонент `Badge` из `@/components/ui/badge`
- Стилизация бейджей должна соответствовать стилю, используемому в компоненте `PostsTable`
- Необходимо сохранить ограничение на количество отображаемых тегов (3 + счетчик)
- Необходимо убедиться, что изменение не влияет на производительность отображения таблицы
