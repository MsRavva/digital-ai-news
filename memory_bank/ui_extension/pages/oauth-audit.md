# OAuth Audit Page

## Маршрут

- `/profile/oauth-audit`

## Назначение

- Отдельная страница для ролей `teacher` и `admin`, где показываются последние OAuth-сессии GitHub/Google независимо от результата.

## Источники данных

- Серверная проверка пользователя и роли через Supabase session + таблицу `profiles`.
- Основной источник списка: таблица `oauth_audit_logs`, читаемая на сервере через admin client после проверки прав.

## UI-состав

- Сводка по количеству успешных, ошибочных и активных OAuth-flow.
- Список последних логов с provider, source (`login`/`register`), шагом, redirect target и итоговым сообщением.
- Отдельные блоки с диагностическими сообщениями и хронологией шагов каждой сессии.

## Навигация

- Ссылка находится в dropdown-меню профиля и показывается только для `teacher`/`admin`.
