# Product Context

## Для чего нужен продукт

Платформа нужна для централизованного доступа к AI-новостям и внутреннему контенту, где пользователи могут читать, создавать и редактировать материалы в зависимости от роли.

## Основные пользовательские сценарии

- Войти через email/password или OAuth.
- Вернуться на исходную страницу после авторизации.
- Просматривать ленту и архив публикаций.
- Создавать и редактировать материалы.
- Управлять профилем пользователя.
- Выполнять административные действия при наличии прав.

## Текущая пользовательская боль

- Нестабильный редирект после авторизации и OAuth ухудшает UX и вызывает ложные переходы на главную вместо исходной страницы.

## Критерий успеха

- После любой авторизации пользователь попадает на ожидаемый безопасный внутренний маршрут.
- Query string целевого URL не теряется.
- Поведение одинаково для password login, регистрации и OAuth.
