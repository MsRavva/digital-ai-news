# Правила проекта Digital AI News

## Общие правила работы

- Отвечать на русском языке и следовать инструкциям пользователя.
- Использовать пакетный менеджер **bun** для всех операций с зависимостями и скриптами.
- Всегда проверять файлы после внесения изменений на линтинг с помощью **biome** и автоматически исправлять их.
- Использовать команды терминала в формате **PowerShell**, так как пользователь работает на Windows.

## Сервер разработки

- **Не проверять статус, не запускать, не останавливать!** Сервером разработки управляет пользователь.
- Сервер запущен на порту 3000 и не требует перезапуска.

## Работа с файлами

- Не спрашивать разрешения перед просмотром файлов проекта, а сразу использовать инструменты для просмотра.
- Не предлагать код для нескольких вариантов решения. Автоматически выбирать лучшее на основе мировых практик и применять его автоматически.

## Изменения в коде

- Никогда самостоятельно не изменять что-либо, кроме того что запланировано или попросил пользователь.
- Предлагать дополнительные улучшения в конце отчета о выполненной задаче.
- При изменении элементов стиля всегда учитывать текущий вариант темы (темный/светлый) и изменять второй соответствующим образом.

## Работа с Git и документацией

- Работа с git и документацией только по прямой команде пользователя.
- Не коммитить изменения на Github самостоятельно. Только по команде пользователя!

## Браузер

- Не открывать дополнительных вкладок браузера.

## Обработка ошибок

- Если не можешь исправить ошибку за две попытки:
  1. Прекратить попытки
  2. Провести детальный анализ почему действия не дали результата
  3. Предложить альтернативные способы решения проблемы

## TypeScript

- Никаких JS! Язык приложения **TypeScript**!
- Для запуска скриптов используй синтаксис: `npx tsx <имя скрипта>`
