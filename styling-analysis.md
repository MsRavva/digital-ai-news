# Анализ стилизации проекта

## Обзор текущей реализации

Проект использует комбинацию Tailwind CSS и кастомных CSS-классов для стилизации. Основные технологии:
- Tailwind CSS как основной фреймворк
- CSS-переменные для темизации
- class-variance-authority для компонентов с вариантами
- next-themes для управления темами
- Shiki для подсветки синтаксиса кода

## 1. Использование Tailwind CSS и кастомных классов

### Текущее состояние:
- Проект активно использует Tailwind CSS для стилизации компонентов
- Есть кастомные классы для специфичных элементов (например, `.saas-button`, `.saas-card`)
- Используются CSS-переменные для темизации (`hsl(var(--primary))`)
- Наблюдается смешение Tailwind классов и кастомных CSS

### Недостатки:
1. **Непоследовательное использование кастомных классов** - некоторые компоненты используют кастомные классы, другие - только Tailwind
2. **Дублирование стилей** - одинаковые стили повторяются в разных компонентах
3. **Сложность поддержки** - кастомные классы разбросаны по разным файлам

### Рекомендации:
1. Создать систему кастомных утилит Tailwind для часто используемых стилей
2. Консолидировать кастомные классы в централизованные файлы
3. Использовать `@apply` директивы для создания переиспользуемых компонентов

## 2. Цветовая палитра и темы

### Текущее состояние:
- Реализована светлая и темная темы через CSS-переменные
- Используется next-themes для переключения тем
- Есть кастомные цвета (saas-purple, saas-blue и т.д.)

### Недостатки:
1. **Несколько разных цветовых схем** - есть стандартные Tailwind цвета и кастомные
2. **Отсутствие четкой иерархии цветов** - не все цвета имеют четкое назначение
3. **Несогласованность в темной теме** - некоторые элементы используют разные цвета

### Рекомендации:
1. Определить четкую цветовую палитру с семантическими названиями (primary, secondary, success, warning, error)
2. Создать систему градаций для каждого цвета (light, base, dark)
3. Убедиться в контрастности для доступности
4. Добавить систему акцентных цветов для интерактивных элементов

## 3. Типографика и читаемость

### Текущее состояние:
- Используется шрифт Mulish
- Есть базовая типографика через Tailwind
- Используется Tailwind Typography plugin для markdown контента

### Недостатки:
1. **Отсутствие четкой типографической системы** - нет четкой иерархии размеров и весов
2. **Несогласованность в компонентах** - разные компоненты используют разные стили для заголовков
3. **Проблемы с читаемостью в темной теме** - некоторые тексты могут быть недостаточно контрастными

### Рекомендации:
1. Создать систему типографики с четкой иерархией (h1-h6, body, caption)
2. Определить базовый размер шрифта и масштабный коэффициент
3. Установить четкие правила для межстрочного интервала и отступов
4. Улучшить контрастность текста в темной теме

## 4. Компоненты UI

3. **Отсутствие документации** - нет четкого описания вариантов компонентов

## Конкретные примеры проблем

1. **Кнопки**:
   - Используются разные стили для одинаковых действий
   - Нет четкой системы размеров кнопок

2. **Карточки**:
   - Разные отступы и стили в разных карточках
   - Несогласованность в вариантах

3. **Типографика**:
   - Заголовки разных компонентов имеют разные стили
   - Нет четкой системы размеров шрифтов
